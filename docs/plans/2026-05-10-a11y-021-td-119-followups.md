1. **Verify dependency landed:** `grep -n "export function wrapAdminError" apps/web/server/supabaseAdmin.server.ts` — must return a hit. If not, halt and wait.
2. **Audit-first commit (READ-ONLY):** Enumerate every call site as a table in the PR description before any code changes — `path:line | error-handling pattern | downstream .code access? | TRPCError? | repository-layer or router?`. Use `grep -rn "supabaseAdmin" apps/web/server/routers apps/web/app/api --include="*.ts"` plus a follow-up grep for `\.code\b` usage on the resulting error variable to flag downstream consumers. Commit this audit as `docs(td-119): admin call-site audit` before touching implementation. Pause for self-review against the audit.
3. **Migration pattern** (apply uniformly):
   - Before:
     ```ts
     const { data, error } = await supabaseAdmin.from('x').insert(...)
     if (error) throw error  // OR throw new Error(error.message) OR throw new TRPCError(...)
     ```
   - After:
     ```ts
     import { supabaseAdmin, wrapAdminError } from "../supabaseAdmin.server"
     // ...
     const { data, error } = await supabaseAdmin.from('x').insert(...)
     if (error) throw wrapAdminError(error)
     ```
   - **TRPCError-wrapping call sites** (e.g. `throw new TRPCError({ code: 'INTERNAL_SERVER_ERROR', cause: error })`): the user-visible message is what reaches the client (tRPC's default `httpBatchLink` serializer drops `cause` — only `message` and `code` survive client-side). To make the env-hint reach the user, hoist the wrapped Error into `message` AND keep the original PostgrestError as `cause` for server-side logging:
     ```ts
     const wrapped = wrapAdminError(error)
     throw new TRPCError({
       code: 'INTERNAL_SERVER_ERROR',
       message: wrapped.message,  // CLIENT-VISIBLE — carries the env-hint
       cause: error,              // SERVER-ONLY — original PostgrestError preserved for Sentry
     })
     ```
     Note explicitly in the inline comment: `cause` is server-side only; the message is what the user sees.
   - **`data`-shape RPC errors** (e.g. `memberships.ts` accept_invite RPC where the error is in `data`, not `error`): these are NOT PostgrestErrors and `wrapAdminError` does not apply. Leave unchanged. The audit table from step 2 must flag these.
   - If the call site does `console.error(error); throw new Error(...)`: keep the console.error for ops visibility AND for Sentry breadcrumb capture, swap the throw to `throw wrapAdminError(error)`.
   - **Downstream `.code` access** (call site or its caller branches on `error.code`): `wrapAdminError` returns a vanilla `Error` with no `code`. Either (a) refactor the downstream branch to read `error.cause?.code` if cause-aware, OR (b) leave the call site unchanged and flag it in a follow-up TD. The audit table from step 2 must surface this; do not silently break a code-aware branch.
4. **Imports:** consolidate the `import { supabaseAdmin } from "../supabaseAdmin.server"` to `import { supabaseAdmin, wrapAdminError } from "../supabaseAdmin.server"` per file. Use Edit's `replace_all` carefully — verify each file still has exactly one import line for the module after the change.
5. **Per-router smoke check:** for each modified router, run its existing test file: `cd apps/web && npx vitest run server/routers/__tests__/<router>.test.ts --reporter=dot`. Any regression halts the sweep.
6. Add ONE new test asserting the error-wrapping path. Project convention: server tests live next to their source under `__tests__/` siblings — verify the right location by running `find apps/web/server -type d -name __tests__`. Existing `wrapAdminError` lives in `apps/web/server/supabaseAdmin.server.ts` (no `apps/web/server/__tests__/` dir at server root today, only `routers/__tests__/` and `repositories/__tests__/`). Create the test file next to its source: `apps/web/server/__tests__/supabaseAdmin.server.test.ts` (and create the `__tests__/` dir if needed). Cover: (a) RLS-class error gets the env-hint prefix, (b) non-RLS error passes through with original message, (c) error with no `code` and no `message` doesn't crash, (d) error with `code === '42501'` triggers the hint regardless of message text, (e) error with `/row-level security/i` in message triggers the hint regardless of code.

**Risk + mitigations:**

- **Risk: a router uses the bare PostgrestError shape (e.g. `error.code` directly) downstream.** Mitigation: `wrapAdminError` returns a vanilla `Error`, not a `PostgrestError`. Any caller that branches on `error.code` will break. Audit each call site for downstream `.code` access before migrating.
- **Risk: TRPCError type contract breaks.** Mitigation: TRPCError accepts `cause: unknown` — the wrapped Error fits. Verify the client-side error-handling boundary still surfaces the message (check the `<TRPCErrorBoundary>` or equivalent).
