## Downstream `.code` access risk

`grep -A3 "if (error)" <files> | grep "error.code"` returned ZERO hits. No call site or its caller branches on `error.code` after the migration would reshape the error. Migration is safe.

## TRPCError pattern decision

Per plan: hoist `wrapAdminError(error)` to `message` (client-visible) and pass original `error` as `cause` (server-only for Sentry). The 7 TRPCError sites become:

```ts
const wrapped = wrapAdminError(error);
throw new TRPCError({
  code: "INTERNAL_SERVER_ERROR",
  message: wrapped.message,  // CLIENT-visible ‚Äî carries env hint
  cause: error,              // SERVER-only ‚Äî original PostgrestError
});
```
