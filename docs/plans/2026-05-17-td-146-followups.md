**Files allowed:**

**Work:**

   - non-UUID pathname (e.g. `/journal/foo`) → items disabled, hint shown, no router.push call
   - non-UUID pathname + menuitem click → addBreadcrumb called once with the failed-validation message
   - PHI check: breadcrumb data has only `matched`/`valid`/`length` — no `pathname`, no raw value, no `rawValue` key at all
