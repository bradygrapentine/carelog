## Design Token Substitution Rules

Apply these substitutions everywhere in target files:

| Old class | New class |
|---|---|
| `bg-white` | remove (Card handles bg) or `bg-card` |
| `bg-gray-50`, `bg-gray-100` | `bg-[var(--color-surface)]` |
| `text-gray-900`, `text-gray-800` | `text-foreground` |
| `text-gray-700`, `text-gray-600` | `text-foreground/80` |
| `text-gray-500`, `text-gray-400` | `text-muted-foreground` |
| `border-gray-100`, `border-gray-200` | `border-border` |
| `rounded-xl` on divs | `rounded-2xl` (card shape) |
| `bg-blue-600`, `bg-blue-700` | `bg-primary` |
| `bg-blue-500` | `bg-primary` |
| `text-blue-600`, `text-blue-700` | `text-primary` |
| `bg-blue-50`, `bg-blue-100` | `bg-[var(--color-primary-subtle)]` |
| `border-blue-200` | `border-[var(--color-border)]` |
| `hover:bg-blue-700`, `hover:bg-blue-600` | `hover:bg-primary/90` |
| `text-red-500`, `text-red-600` | `text-[var(--color-danger)]` |
| `hover:text-red-700` | `hover:text-[var(--color-danger)]/80` |

**Keep unchanged:** `text-green-*`, `text-amber-*`, `text-orange-*`, `bg-green-*`, `bg-amber-*`, `bg-orange-*`, `bg-red-50` (error backgrounds), `text-red-*` inside error messages (these are semantic status colors for mood/alerts).

**Inputs on forms:** Replace bare `border border-gray-200 rounded-lg px-3 py-2 text-sm` with shadcn `<Input />` component from `@/components/ui/input`. Same for `<textarea>` → `<Textarea />` from `@/components/ui/textarea`.

**Buttons:** Replace bare `<button className="bg-blue-600 text-white ...">` with `<Button>` from `@/components/ui/button`. Use `variant="default"` for primary, `variant="outline"` for secondary, `variant="ghost"` for ghost, `variant="destructive"` for destructive.
